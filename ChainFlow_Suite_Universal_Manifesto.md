# ChainFlow Suite ユニバーサル・マニフェスト 📂✍️🔍
## —— 事務実務から生まれた「文書工学（Document Engineering）」の再定義

---

## 1. 序文：なぜ「事務方」がシステムを構築したのか

一般的なソフトウェア開発は「技術的に何が可能か」から出発しがちですが、**ChainFlow Suite** は徹底的に「実務の現場で何が苦痛か」という、ある種の執筆的・事務的情念から誕生しました。

市販のオフィスソフトや、エンジニア向けの洗練されたエディタは、確かに優れています。しかし、日々の膨大な事務作業においては、以下の「断絶」が常に発生していました。

*   **速度の断絶**: 数万のファイルの中から一つを探す、あるいはタブを切り替える際の数秒のラグが、思考のフローを破壊する。
*   **表現の断絶**: Markdownの書きやすさは魅力だが、最終的に提出すべき「公的なPDF書類」にするためのレイアウト調整（印影、改ページ、ミリ単位の配置）が極めて困難である。
*   **環境の断絶**: クラウドへのアップロードを躊躇う機密情報や、複数のPCを使い分ける際のポータビリティの欠如。

ChainFlow Suiteは、これらの断絶を埋め、**「思考の速度でドキュメントを完成させ、公的な成果物として配布する」**ための、事務実務家による事務実務家のための武器です。

## 2. コア・コンセプト：情報の「Flow（連鎖）」

「ChainFlow」という名が示す通り、このスイートは単なる便利ツールの集合体ではなく、一つの「情報の連鎖（パイプライン）」を形成しています。

### ① 収穫・抽出のフェーズ (Sniper V3)
情報の源泉であるWebや業務システムから、必要な「データ」だけを正確に射抜（スナイプ）して収穫する。V3における「4象限抽出（A:Text / Z:Markdown / S:Style / D:DOM）」は、生データから純粋な構造体まで、あらゆる情報の「利用可能な資産化」を可能にしました。

### ② 探索・整理のフェーズ (Search & Filer)
収穫した、あるいは過去に蓄積した膨大な資産から、必要なものに「瞬時」に到達する。OS標準の検索やファイル操作が遅いと感じた瞬間に、実務のスピードは死にます。v22における**I/Oゼロ・アイコン**や**非同期スキャン**といった「極限の高速化」は、単なるスペック競争ではなく、「思考を止めないための最低条件」としての必然です。

### ③ 思考・構築のフェーズ (Writer & Designer)
見つけた情報を元に、構造化し、図解を加える。ここではMarkdownという「構造」と、CSS/HTMLという「表現」をハイブリッドに融合させます。**「Typora以上、DTP未満」**という立ち位置は、事務実務において「書きやすさ」と「形式の厳格さ」を両立させるための黄金比です。

### ④ 完成・提出のフェーズ (PDF Tools)
完成したドキュメントは、PDFという「固定された成果物」として結晶化されます。作成したものを比較し、マージし、あるいはスクラップにする。PDFを「ただの閲覧用ファイル」ではなく、「管理・加工可能な動的データ」として扱う一連のツール群が、実務のラストワンマイルを完結させます。

## 3. 各コンポーネントの意義と立ち位置

### 🔍 ChainFlow Sniper V3 (The Eye / The Harvester)
*   **意義**: 無機質なWebインターフェースから「構造化された意味」を奪還する。
*   **立ち位置**: 外部情報の玄関口。iFrameやShadow DOMを透過する「深層分解」に加え、CSS装飾を極限まで復元する「Sモード」と、AI連携に特化した純粋構造抽出「Dモード」を使い分ける、情報のマルチツール。

### 📁 ChainFlow Filer (The Control Tower)
*   **意義**: ファイル操作を「OSの管理下」から「ユーザーの思考下」へ奪還する。
*   **立ち位置**: 全てのタスクの起点。マウスを握る時間を削り、キーボードのWASD/Vimライクな移動で「ディレクトリの深海」を縦横無尽に泳ぐための潜水艦。

### 🔍 ChainFlow Search (The Oracle)
*   **意義**: 記憶の外部ストレージに対する超高速検索。
*   **立ち位置**: 「あのファイル、どこだっけ？」という自問自答を過去にする、Suiteアクセシビリティの基盤。ワイルドカード検索による一括キャッシュ構築や、複雑な条件式、日付順の並べ替えなど、単なる「検索」を超えた「情報のハンドリング環境」を提供します。

### ✍️ ChainFlow Writer v9 (The Factory)
*   **意義**: Markdownの構造美を、公的なPDFの様式美へ変換する。
*   **立ち位置**: 単なるテキストエディタではなく、「提出書類生成エンジン」。v9での「IDE-Style Syntax Highlighting」は、複雑なHTML/CSSノイズと人間の思考（本文）を視覚的に分離。リアルタイム改ページとスタンプ構文（印影配置）は、事務職における究極の武器となります。

### 📄 PDF Studio / Compare / Scrap (The Finisher)
*   **意義**: PDFを不可逆な「紙」から、可逆な「構成部品」へ戻す。
*   **立ち位置**: 業務上の検証と品質保証。差分をミリ単位で検出し、必要な部分を削り取る「外科医のメス」。

## 4. 技術的・思想的な独自基盤

### ● Local-First & Portable (所有と自由)
Cloud-nativeが全盛の時代に、敢えて「ローカル実行・ポータブル構成」に拘っています。これは、レジストリを汚さず、USB一つで自分の城（環境）をどこでも展開できるという「職人の道具箱」へのオマージュです。

### ● Performance as UX (速さは正義)
v22で追求された「I/O負荷の最小化」や「共有モデルによるメモリ節約」は、低スペックな事務用PCでも「プロの道具」として機能させるための技術的配慮です。

### ● Semantic Extraction Engine (知能的な抽出)
Sniper V3に搭載された「A/Z/S/D 4象限抽出モデル」は、単なる座標コピーを卒業。DOMの構造や属性名から「それがデータであるかどうか」をAI的に評価・選別（Dモード）する、次世代の事務インフラとしての核となります。また、CSS GridやFilterまでをもポータブルに再構築するSモードは、デザインの資産価値を損なわない。

### ● Chromium + Qt + Python (黄金の三角形)
Pythonの柔軟性、Qtの堅牢なUI、Chromiumの圧倒的な表現力。これらを組み合わせることで、OSの枠を超えた「独自のドキュメント環境」を実現。具体的には、**PySide6 (Qt for Python)** を核とし、以下の高度な統合を行っています：
*   **QWebEngineView × QWebChannel**: JavaScriptとPython間のシームレスな双方向スルーパス。
*   **IDE-Style Editor (Multi-Layer Regex)**: 正規表現のLookaroundを活用し、大規模なインラインスタイルを含むHTMLを、パフォーマンスを落とさず多色色分けする高度な描画ロジック。
*   **Custom QSS (Qt Style Sheets)**: ベクターベースで設計された、高解像度（DPI-aware）かつモダンなダークモードUI。
*   **Event Filter & Native API**: Windowsのシグナルを直接捕捉し、タイトルバーの描画（DWM制御）までをもプログラマブルに制御。
*   **Signals & Slots Architecture**: コンポーネント間の疎結合な状態同期。

「地味で目立たないが、一瞬で仕事が終わり、成果物は誰よりも美しい」
