# ChainFlow Writer v6 記述・生成仕様書 (for AI Assistants)

このドキュメントは、**ChainFlow Writer v6** に最適化された Markdown コンテンツを生成・執筆するために AI アシスタントが遵守すべき仕様をまとめたものです。

---

## 1. 基本的な設計思想
ChainFlow Writer は「Markdown の手軽さ」と「DTP (卓上出版) のレイアウト自由度」の融合を目指したエディタです。標準的な Markdown 記法に加え、独自の拡張記法を備えています。

## 2. 最重要：レイアウトとインデント
- **左詰め原則**: Markdown ベースであるため、HTML タグの内側であってもコンテンツは基本的に**左詰め（インデントなし）**で記述することを推奨します。
- **デデント機能**: 本アプリは HTML タグ内の Markdown をレンダリングする際、タグの階層に基づいたインデントを自動的に無視（デデント）します。

## 3. 独自拡張機能

### 3.1 `<m-d>` タグ (Markdown in HTML)
HTML の `<div>`, `<table>`, `<td>` 等の内部で Markdown を有効にするための独自タグです。
- **用途**: 「複雑な表のセル内で箇条書きや数式を使いたい」場合に必須です。
- **例**:
  ```html
  <td>
    <m-d>
    - 箇条書き1
    - 箇条書き2
    </m-d>
  </td>
  ```

### 3.2 変数埋め込み (`{{key}}`)
フロントマターで定義された変数を本文中で呼び出せます。
- **記法**: `{{variable_name}}`
- **適用範囲**: 本文、Mermaid 図解内、LaTeX 数式内、属性値。

### 3.3 特殊ブロック (`:::`)
コンテナ型の装飾・レイアウト指定です。
- `::: center` / `::: right`: 配置指定
- `::: large` / `::: small`: 文字サイズ
- `::: info` / `::: warning`: アドモニション（警告・情報）
- `::: stamp`: 画像の絶対配置（電子印影等）
- `::: no-print`: プレビューには出すが PDF 出力には含めない

## 4. 特殊レンダリング仕様

### 4.1 Mermaid v11 (ESM)
最新の Mermaid v11.4 を搭載。従来の図解に加え、`xychart-beta` などの最新記法が利用可能です。
- 生成時は `mermaid.init()` を呼び出す古い記法ではなく、最新の仕様に準拠したコードを推奨します。

### 4.2 LaTeX (KaTeX v0.16)
高精度な数式描画をサポート。
- `$ $` (インライン) および `$$ $$` (ブロック) が利用可能。

### 4.3 画像のリサイズ
Markdown の画像記法に続けて `{width=50%}` 等を記述することで、CSS ベースではない物理的なレンダリングサイズを指定できます。
- 例: `![alt](image.png){width=300px}`

## 5. PDF 出力への配慮
- **改ページ制御**: `<div style="page-break-before: always;"></div>` を挿入することで、PDF 出力時にも確実に改ページされます。
- **Fidelity**: プレビューで見えているレイアウト（ページ境界線等）がそのまま PDF に反映される設計です。

## AI への指示例 (Prompt Example)
> 「あなたは ChainFlow Writer v6 のエキスパートです。添付の仕様書に基づき、`<m-d>` タグや `::: center` ブロックを適切に使い、プロフェッショナルな報告書を Markdown で生成してください。インデント崩れに注意し、コンテンツは基本的に左詰めで出力してください。」
