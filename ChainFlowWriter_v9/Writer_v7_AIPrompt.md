# ChainFlow Writer v7 記述・生成仕様書 (for AI Assistants)

このドキュメントは、**ChainFlow Writer v7** に最適化された Markdown コンテンツを生成・執筆するために、AI アシスタントが遵守すべき仕様をまとめたものです。

---

## 1. 基本的設計思想
ChainFlow Writer は「Markdown の手軽さ」と「DTP (卓上出版) のレイアウト自由度」の融合を目指したエディタです。標準的な Markdown 記法に加え、独自の拡張記法を備えています。

## 2. 最重要：レイアウトとインデント
- **左詰め原則**: Markdown ベースであるため、HTML タグの内部であってもコンテンツは基本的に**左詰め（インデントなし）**で記述することを推奨します。
- **デデント機能**: 本アプリは HTML タグ内部の Markdown をレンダリングする際、タグの階層に基づいたインデントを自動的に無視（デデント）します。

## 3. 独自拡張機能

### 3.1 `<m-d>` タグ (Markdown in HTML)
HTML の `<div>`, `<table>`, `<td>` 等の内部で Markdown を有効にするための独自タグです。
- **用途**: 複雑な表のセル内部で箇条書きや数式を使いたい場合等に必須です。
- **例**:
  ```html
  <td>
    <m-d>
    - 箇条書き
    - 箇条書き
    </m-d>
  </td>
  ```

### 3.2 変数埋め込み (`{{key}}`)
フロントマターで定義された変数を本文内で呼び出せます。
- **記法**: `{{variable_name}}`
- **適用範囲**: 本文、Mermaid 図解内部、LaTeX 数式、属性値。

### 3.3 特殊ブロック (`:::`)
コンテナ型の装飾・レイアウト指定です。
- `::: center` / `::: right`: 配置指定
- `::: large` / `::: small`: 文字サイズ
- `::: info` / `::: warning`: アドモニション（警告・情報ボックス）
- `::: stamp`: 画像の絶対配置。電子印影等に最適。
- `::: no-print`: プレビューには出すが PDF 出力には含めない執筆用メモ。

## 4. 特殊レンダリング仕様
### 4.1 Mermaid v11 (ESM)
最新の Mermaid v11.4 を搭載。`xychart-beta` などの最新記法が利用可能です。

### 4.2 LaTeX (KaTeX v0.16)
高精度な数式描画をサポート。`$ $` (インライン) および `$$ $$` (ブロック) が利用可能です。

### 4.3 PDF 出力への配慮 (Fidelity Sync)
- **強制改ページ**: `<div style="page-break-before: always;"></div>` を挿入することで、PDF 出力時にも確実に改ページされます。
- **マージン相殺対策**: v7 では `padding-top: 0.1mm` 等の内部処理により、改ページ直後の見出しの余白が保護されます。

## AI への指示例 (Prompt Example)
