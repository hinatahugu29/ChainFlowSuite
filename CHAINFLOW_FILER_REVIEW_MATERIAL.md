# ChainFlow Filer v23.2 - System Overview & Review Material

このドキュメントは、**ChainFlow Filer v23.2 [Modular Refined]** のアーキテクチャ、設計思想、技術スタック、および進捗状況をまとめたものです。
外部のAIアシスタントにこのアプリケーションのコンテキストを共有し、コードレビューやアーキテクチャ設計のアドバイスを受けるための「プロンプト入力用素材」として作成されています。

---

## 1. アプリケーションの概要 (Executive Summary)

ChainFlow Filer は、Python と Rust を融合させたプロフェッショナル向けの高度なファイルマネージャーです。
単なる「フォルダ閲覧ソフト」ではなく、大量の素材を複数のツール群へ高速かつ正確に「流し込む（Flow）」ための**「ワークフローのハブ（作業用コックピット）」**として設計されています。

### コアコンセプト・独自性
- **Native Speed (v23.1+)**: ディレクトリ走査やソート処理を Rust (PyO3) で実装したネイティブコアへ委譲し、数万ファイル規模のフォルダも瞬時にハンドリング。
- **Modular Architecture (v23.2+)**: 肥大化していた UI クラスからビジネスロジック（ファイル操作、セッション管理）を分離し、保守性と堅牢性を大幅に向上。
- **Flow UI**: 画面を複数のペインに分割し、素材の移動や比較をキーボード基準の操作（H/J/K/L, Q, S, X 等）で完結させる独自の操作体系。
- **Stability Guard**: `shiboken6` を利用した C++ オブジェクトの生存確認により、高速操作時のクラッシュ（RuntimeError）を徹底排除。

## 2. 技術スタック (Tech Stack)

- **言語**: Python 3.11+ / Rust (PyO3)
- **GUI フレームワーク**: PySide6 (Qt for Python 6)
- **ネイティブエンジン**: `chainflow_core` (Maturin によりビルドされた Rust モジュール)
- **ビルドツール**: Nuitka (Python -> C 変換による高速化・難読化)
- **非同期処理**: `QThread` (Rust エンジンとの非同期通信)
- **安定性レイヤー**: `shiboken6` (C++/Python オブジェクト生存チェック)

---

## 3. 主要なモジュール構造 (Module Architecture)

### 階層構造
1. **`main.py`**: エントリーポイント。バージョン管理、サンドボックス環境の判定、多重起動防止などを担当。
2. **`widgets/main_window.py`**: 画面全体の統括、グローバルショートカット、フロー（タブ）管理。
3. **`widgets/file_pane/file_pane.py`**: フォルダ表示の最小単位。
4. **`core/actions.py`**: [v23.2 新設] `FilePane` から分離された実質的なファイル操作ロジック（コピー、リネーム、ツール起動等）。
5. **`core/session.py`**: [v23.2 新設] タブやペイン、ソート順などの永続化（保存・復元）を担当するセッションマネージャー。

### データモデル
- **`models/native_model.py` (NativeFileModel)**: 
  - `QFileSystemModel` を廃止し、Rust 側の `list_directory_native` から取得したデータを保持する独自リストモデル。
  - OS への I/O を Rust レイヤーに閉じ込めることで、描画性能を極限まで高めている。

## 4. 進捗状況と解決済みの課題 (Milestones & Fixes)

### 1. [v23.2] FilePane クラスの解体 (Modularization)
- **状況**: 部分的解決。
- **内容**: 以前の「God Object」化していた `FilePane` から、ファイル操作ロジックを `core/actions.py` へ、セッション管理を `core/session.py` へ切り出し。
- **メリット**: UI 定義とロジックが分離され、単体テストや機能拡張が容易になった。

### 2. [v23.2] RuntimeError (C++ object already deleted) の根絶
- **状況**: **解決済み**。
- **内容**: タブ切り替えやフォルダ遷移の瞬間に古いウィジェットへアクセスする競合を防ぐため、`shiboken` による `isValid` チェックを全ての走査ループに導入。

### 3. [v23.1] パフォーマンスの劇的向上
- **状況**: **解決済み**。
- **内容**: 標準の Python ライブラリによる I/O を Rust エンジンへ移行。検索やソートがネイティブ速度で実行される。

## 5. 今後の技術的焦点 (Future Targets)

AI やシニアエンジニアにレビューを依頼する際の現在の焦点は以下の通りです。

1. **Rust-Python 間のブリッジ最適化**:
   大量のファイル情報を渡す際のシリアライズオーバーヘッドの最小化アプローチ。
2. **更なるロジック分離**:

   `tools.json` 形式を拡張し、他アプリケーションが ChainFlow の「フロー」に自律的に参加するための標準プロトコルの策定。

## 7. 外面的特徴とユーザー体験 (Visual & Interaction)

技術的な堅牢性だけでなく、ChainFlow Filer は「道具としての手馴染み」を追求した外面的特徴を備えています。

### 「Flow UI」のレイアウト構成：超タイル型・情報集約コックピット
本ファイラは単なるリスト表示の枠を超え、数千のファイル群を掌掌握するための**「情報コックピット」**として機能します。

- **Massive Parallel Tiling (超多ペイン同時並置)**: 
  単一のウィンドウ内に、無数のディレクトリをタイル状に敷き詰めることが可能です。幅の狭いレーンを多数並べ、それぞれの中に3〜4つのペインを**「縦スタック（Vertical Stack）」**させることで、関連する複数のプロジェクトフォルダを一元的に監視・操作できます。
- **Dynamic Orientation (変幻自在なレーン構成)**: 
  特定の作業に集中するための「広大なボトムペイン」と、その流し込み元となる「密集したアッパーペイン」を組み合わせるなど、ワークフローに合わせてレイアウトを動的に最適化できます。
- **Persistence (意図しない「閉じ」の排除)**: 
  一度構築したこの高密度な「布陣（レイアウト）」は、ユーザーが明示的に閉じない限り維持されます。OS標準ファイラのように「フォルダ移動のたびに前の表示が消える」といった思考の断絶は一切ありません。
- **「従兄弟フォルダ問題」の構造的解決**: 
  ツリー構造の末端に点在する「従兄弟」のような遠いフォルダ同士を、レーン操作によって目の前に引き寄せ、最短距離で接続します。
- **Ultra-High Information Density**: 
  1800x950 以上の大画面をフルに活用し、装飾を極限まで削ぎ落としたミニマルデザインにより、プロフェッショナルの視線移動を最小限に抑えます。

### インタラクション・モーダル（操作感）
マウスを「補助」に追いやり、キーボードを「主役」に据えたプロ向けのエディタに近い操作体系を強調しています。
- **Keyboard-Driven Workflow**: GUIでありながら、全ての基本操作をホームポジションから離れずに実行可能。マウス操作による「思考の断絶」を物理的に排除しています。
- **Instant Action Keys**: Q（終了/閉じる）、S（検索/選択）、X（実行/削除）、V（垂直分割）など、単キーまたは最小限のコンビネーションによるコマンド実行。
- **Slash Menu (Command Palette)**: `Ctrl + P` により、アプリケーション全体の機能を曖昧検索から即座に呼び出し可能。

### ステータス・インジケータと描画レスポンス
高度な処理をストレスなくユーザーに伝えるための視覚的フィードバックを備えています。
- **超高速描画レスポンス**: Rust エンジンによる「0.01秒」の描画レスポンス。数万件のファイルがあってもスクロールやフォルダ遷移時の「待ち時間」がゼロに近く、吸い付くような操作感を実現。
- **Parallel Progress Bar**: ファイル操作（コピー・移動等）や検索処理において、マルチコア稼働状況を反映したプログレス表示を行い、バックグラウンドでの並列処理を可視化。
