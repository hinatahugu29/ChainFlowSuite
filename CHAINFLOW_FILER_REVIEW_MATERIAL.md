## 1. アプリケーションの概要 (Executive Summary)

ChainFlow Filer は、Python (PySide6) で開発されているプロフェッショナル向けの高度なファイルマネージャーです。
一般的な「フォルダを閲覧する」ためのエクスプローラーとは異なり、複数の素材（ファイル）を様々なツール群へ「流し込む（Flow）」ための**「ワークフローのハブ（作業用コックピット）」**として設計されています。

### コアコンセプト・独自性
- **Flow アーキテクチャ**: 画面を複数のペイン（枠）に縦横無尽に分割し、左から右（または上から下）へ作業工程を連ねていくUI。
- **キーボード＆ホバー・ドリブン**: 「マウスをホバーするだけでペインのフォーカスが切り替わる」「`Ctrl+P` でコマンドパレットを呼ぶ」など、入力デバイス間の無駄な移動を排除。
- **シームレスなツール連携 (ChainFlow Suite)**: 同期開発されている DTPエディタ（ChainFlow Designer）や PDFツール（ChainFlow PDF Studio）などの子アプリケーションをプラグイン経由で呼び出せる設計。

## 2. 技術スタック (Tech Stack)

- **言語**: Python 3.x
- **GUI フレームワーク**: PySide6 (Qt for Python 6)
- **ビルドツール**: PyInstaller (Windows環境の`.exe`化、バッチスクリプトによる自動化)
- **非同期処理**: `QThreadPool`, `QRunnable` (ファイル操作、アイコン読み込み等)
- **アーキテクチャパターン**: Model-View パターン (Qtの標準機能に準拠)

## 3. 主要なモジュール構造 (Module Architecture)

### UI階層構造
巨大な画面内レイアウトを効率よく管理するため、段階的なコンテナクラスを使用しています。

1. **`main_window.py` (ChainFlowFiler)**: エントリーポイント。QMainWindowの継承。アドレスバー、サイドバー（NavigationPane）、タブ（フローエリア）を統括。
2. **`widgets/flow_area.py` (FlowArea)**: 1つのタブ内全体を管理。
3. **`widgets/lane.py` (HorizontalLane / VerticalLane)**: 分割された直線状のエリア。中に複数のPaneを保持。
4. **`widgets/file_pane/file_pane.py` (FilePane)**: 実際のファイルリストを表示するメインユニット。一番重いクラス。
5. **`widgets/file_pane/batch_tree_view.py` (BatchTreeView)**: QTreeViewをオーバーライドし、独自のドラッグ＆ドロップ動作やホバーフォーカス機能を追加。

### コア機能・データ処理
- **`models/proxy_model.py` (SmartSortFilterProxyModel)**: `QFileSystemModel` のラッパー。ディレクトリを優先的に上に表示するカスタムソート機能や、非同期アイコン結合、複数キーワードのインクリメンタルサーチ（フィルタリング）を担う。
- **`core/icon_loader.py` (IconLoader / IconWorker)**: スレッドプールを使ったアイコンの非同期取得機構。UIスレッドのブロッキングを防ぐ。
- **`core/file_worker.py` (FileOperationWorker)**: 巨大ファイルのコピー/移動/ZIP圧縮解凍をバックグラウンドスレッドで実行する機構。
- **`core/plugin_manager.py` (PluginManager)**: `tools.json` に定義された外部ツールキット（ChainFlow Designer等）をサブプロセスとして起動・管理するハブ機能。

## 4. ユニークな機能群 (Unique Features)

- **マーク＆バッチ操作 (Mark/Collect)**:
  - 異なるディレクトリにあるファイルを「お気に入り（カゴ）」に追加するように一時マークし、別ツールや別フォルダへ一括でドラッグ＆ドロップできる。
- **スラッシュメニュー / コマンドパレット (`slash_menu.py`)**:
  - VSCodeのようなポップアップコマンドメニュー（`Ctrl+P` で起動）。ペインの分割、新しいツールの起動、設定の呼び出しなどをキーボードのみで完結。
- **Quick Look (`quick_look.py`)**:
  - ファイル選択時にスペースキーを押すだけで、テキストや画像の内容を瞬時にポップアッププレビューする機能（macOSのQuickLookインスパイア）。
- **セッション復元**:
  - ウィンドウサイズ、ペインの分割状態、開いていたディレクトリパスを終了時に記憶し、再起動時に完全復元。

## 5. 現在抱えている技術的課題・ボトルネック (Known Issues / Refactoring Targets)

アーキテクチャ設計やコード品質に関して、他のAIやシニアエンジニアにレビューしてもらう際の主な焦点は以下の点です。

### 1. FilePaneクラスの肥大化 (God Object Anti-pattern)
- `file_pane.py` が非常に大きく（千数百行レベル）、UI管理・ファイル操作（コピーパーストの判定）・コンテキストメニュー構成・イベントハンドリングのすべてを内包している。
- **レビュー希望点**: MVC/MVPパターン等への分離、あるいはストラテジーパターン等を用いた責務の切り出しアプローチ。

### 2. QuickLookのメインスレッドブロッキング
- QuickLookプレビュー時に、巨大な画像ファイルのデコードや、数万ファイルを含むディレクトリの `os.listdir()` 実行処理がUIスレッドに乗っており、一瞬フリーズするリスクがある。
- **レビュー希望点**: PySide6 における軽量な非同期プレビュー描画の設計手法（キャンセル処理などのスマートな実装方法）。

### 3. ProxyModel と表示パフォーマンス
- QFileSystemModelをベースにしているため、深いディレクトリ構造（`setRecursiveFilteringEnabled(True)` 時）で検索をかけると、OSへのI/O負荷と文字列マッチング処理（Pythonレイヤー）が重なり、タイピング時のもたつき（遅延）が発生しやすい。
- **レビュー希望点**: Qtにおける億単位のファイルパスの高速インメモリー検索とモデルツリーのハイブリッドな構築手法。状態管理のキャッシュ最適化。

### 4. ネットワークドライブへの配慮
- 古いネットワークマウントを跨ぐ際、変更検知（QFileSystemWatcher）に頼り切れず、意図的に重いルート再設定（Refresh）を強制している挙動がある。
