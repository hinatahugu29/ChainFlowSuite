## 2. アプリケーション別・開発規模詳細

### ① ChainFlow Writer (v9) —— 【究極の文書工学エンジン】
* **開発規模**: **17.5 人月**
* **技術的 特異点**:
    * **「IDE-Style Multi-Layer Syntax Highlighting」**: v9にて実装。正規表現のLookaround（先読み・後読み）を活用した、多層パースエンジン。数千行に及ぶインラインスタイルを含むHTMLタグを低コントラスト化しつつ、タグ名や属性値を意味的に色分け。物理的なデータ可読性を損なわずに、ユーザーの認知負荷を激減させる「UI心理学」に基づく実装。
    * **「リアルタイム A4 改ページエンジン」**: `preview_pane.py` に実装された、物理的なA4サイズに基づき編集中のDOMを動的にページ分割する独自ロジック。
    * **魔法のタグ `<m-d>`**: HTMLコンテナ内でMarkdownを直接パース・描画可能にする独自拡張。
    * **Stamp Syntax & Blend Modes**: `::: stamp` 構文による絶対座標配置。デジタル印影にリアルな品質を提供。
    * **1:1 PDF Fidelity & Path Normalizer**: Chromium（QWebEnginePage）レンダリングの物理サイズ完全一致の保証。

### ② ChainFlow Filer (v22) —— 【I/O負荷ゼロの高速基盤】
* **開発規模**: **8.5 人月**
* **技術적 特異点**:
    * **I/Oゼロ・アイコン描画エンジン**: `proxy_model.py` において、`QIdentityProxyModel` を継承し拡張子に基づきインメモリ（QPixmapCache）でアイコンを動的生成。数万件のファイル表示時もディスクI/Oを発生させない究極の最適化。
    * **グローバル・シングルトン・システムモデル**: `QFileSystemModel` を継承した独自の共通モデルを全ペインで共有。`setData` や `directoryLoaded` シグナルの効率的な集約。
    * **Incremental Refresh & Debounced Updates**: `proxy.invalidate()` による差分更新と、`QTimer` によるマウスホバー時のオートフォーカス速度を制御するデバウンス処理の実装。
    * **DWM API 直接制御**: `main_window.py` にて `ctypes` を介してWindowsネイティブAPIを直接制御。`DwmSetWindowAttribute` 等によるタイトルバーと枠線の完全なダークモード化。

### ③ ChainFlow Designer —— 【幾何学的知能を備えたD2D（DTP to Document）】
* **開発規模**: **7.5 人月**
* **技術的特異点**:
    * **Smart Snapping Engine**: `QGraphicsScene` 上でオブジェクトの座標を監視し、`itemChange` イベントフックによりミクロン単位での吸着・幾何学計算（Scene座標系）をリアルタイムに実行。
    * **Pro Table Widget**: `QGraphicsProxyWidget` を介してQTableWidgetを配置するのではなく、`QGraphicsItem` として自作することで、Excel風の行・列ドラッグリサイズとズームに耐える描画品質を両立。
    * **QUndoStack ベースのコマンドパターン**: 全てのキャンバス操作を `QUndoCommand` としてカプセル化。大規模なレイアウト変更も瞬時に元に戻せる堅牢なアンドゥ・リドゥシステム。
    * **XML/SVGベースの厳密な座標永続化**: 拡大・回転（Matrix）・グループ化を行っても誤差が蓄積されない、数学的に厳密なシリアライズロジック。

### ④ ChainFlow Sniper (V3) —— 【深層データ・UI抽出エンジン】
* **開発規模**: **8.5 人月**
* **技術적 特異点**:
    * **D-Mode (Pure DOM Snipe)**: AI連携およびWriterテーマ同化に特化した、純粋セマンティック構造抽出。全ての装飾プロパティと寸法属性をパージしつつ、論理構造のみを100%の純度で抽出する独自クリーニング・アルゴリズム。
    * **S-Mode Enrichment (見た目の完全復元)**: CSS Grid, background-size/position, transform, filter などの高度なプロパティをポータブルに再構築。同時に、抽出したHTMLを物理的なハミ出しから保護する「堤防ラッパー（Breakwater Wrapper）」ロジックにより、A4紙面の整合性を保証。
    * **深層貫通探索 (Deep Penetration)**: `QWebEnginePage.runJavaScript` による再帰的なインジェクション。iFrame、Shadow DOMの内部、および透明なオーバーレイに隠されたデータを正確に捕捉。
    * **A/Z/C/S/D Hold-To-Activate UX**: 抽出モードごとに色分けされた直感的なフィードバック（Sniper-Highlight-DOM等）。マウスホイールによるDOM木（Treeレベル）の動的ズーム機能。
    * **QWebChannel 通信ブリッジ**: JSONシリアライズのオーバーヘッドを最小化した、抽出データとQt側の高速同期。

### ⑤ ChainFlow Search —— 【情報の高速道路】
* **開発規模**: **4.5 人月**
* **技術的特異点**:
     * **Hybrid Async Engine**: `QRunnable` と `QThreadPool` を用いたバックグラウンド高速スキャン。UIスレッドを一切ブロックせず、数万件のインデックスをメモリ上に構築。
     * **Advanced Query Stream**: 自作のパーサーにより、メモリ内でのAND/OR/Wildcard検索をストリーム処理で実現。
     * **IPC的連携エコシステム**: `QLocalSocket` や独自のURIスキームを用いた、検索結果からFilerやWriterへの即座なデータ引き渡し。

### ⑥ PDF Suite & Common Infrastructure (Studio/Compare/ToDo/Tool)
* **開発規模**: **6.0 人月**
* **技術的特異点**:
    * **PDF 1-Dot Comparison**: `compare.py` 等に実装された、同期スクロールによる精密な比較と差分ハイライト。
    * **共通ポータビリティ基盤**: カレントパス解決、AppUserModelID統合、およびポータブルJSONデータベースによる環境非依存の実装。
